# Макет Google Таблицы

## Генерируемый план

Для настройки фактической закрытой Google Таблицы используется генерируемый план:

```text
build/google-sheet-tabs-plan.md
build/google-sheet-tabs-plan.csv
```

Создать его можно командой:

```bash
make tabs-plan
```

План строится по `scripts/tools/workbook_config.py`, поэтому совпадает с XLSX-сборкой и планом импорта.

## Лист 1. Главная

Назначение: быстрый вход в базу.

Блоки:

- быстрые ссылки на разделы;
- самые частые ситуации;
- последние изменения;
- новые предложения сотрудников;
- ответственные за разделы.

## Лист 2. Контакты

Основной лист для организаций и партнёров.

Ключевые поля:

- ID;
- тип;
- категория;
- подкатегория;
- организация;
- контактное лицо;
- телефон;
- email;
- адрес;
- когда обращаться;
- документы;
- практический совет;
- частые ошибки;
- приоритет;
- статус.

## Лист 3. Ситуации

Самый полезный лист для новичка.

Ключевые поля:

- ситуация;
- кратко;
- что делать;
- куда обращаться;
- документы;
- риски;
- частые ошибки;
- связанные контакты.

## Лист 4. База знаний

Документы, инструкции, алгоритмы и шаблоны.

Ключевые поля:

- ID;
- тип;
- название;
- когда использовать;
- пошагово;
- связанные контакты;
- связанные документы;
- статус.

## Лист 5. Практика

Короткие заметки сотрудников.

Ключевые поля:

- дата;
- автор;
- раздел;
- ID записи;
- практический совет;
- полезность;
- статус.

## Лист 6. Предложения изменений

Входящий поток правок.

Ключевые поля:

- дата;
- автор;
- раздел;
- ID записи;
- что изменить;
- что было;
- что стало;
- почему важно;
- статус.

## Лист 7. Справочники

Технический лист для выпадающих списков:

- категории;
- типы объектов;
- статусы;
- приоритеты;
- роли.

## Правило интерфейса

Для сотрудников таблица должна быть простой. Технические поля можно скрывать, если они мешают работе.
