# Критерии готовности MVP

## Назначение

Этот документ помогает понять, когда базу можно считать не просто черновиком, а рабочим MVP для офиса.

## Минимальная готовность структуры

MVP готов по структуре, если есть:

- листы для контактов, ситуаций, базы знаний, FAQ, практики и предложений;
- справочники категорий, статусов и приоритетов;
- единый принцип ID;
- правила заполнения;
- правила безопасности;
- план обслуживания.

## Минимальная готовность данных

В рабочей закрытой таблице должно быть:

- не менее 30 проверенных или частично проверенных контактов;
- не менее 10 типовых ситуаций;
- не менее 10 базовых документов;
- не менее 10 FAQ;
- не менее 5 практических заметок;
- назначены ответственные за ключевые разделы.

## Минимальная готовность для сотрудника

Сотрудник должен уметь:

- найти ситуацию;
- найти контакт;
- понять статус записи;
- перейти к связанному документу;
- предложить правку;
- не путать публичный репозиторий и закрытую рабочую таблицу.

## Тестовые задания

MVP считается рабочим, если 2–3 сотрудника выполняют задания:

1. Найти порядок действий при сделке с детскими долями.
2. Найти порядок действий при утере документа.
3. Найти документ для проверки объекта.
4. Найти контактный раздел по оценке для банка.
5. Найти, как предложить правку.

## Измеримые критерии

- Ответ по типовой ситуации находится за 1–3 минуты.
- Не менее 70% тестовых заданий выполняется без подсказки менеджера.
- После теста собрано не менее 5 полезных замечаний.
- Нет критичных ошибок безопасности.
- Непроверенные записи имеют статус `CHECK` или `DRAFT`.

## Когда нельзя запускать MVP

Не запускать на весь офис, если:

- сотрудники не понимают структуру;
- нет ответственного за обслуживание;
- в таблице смешаны проверенные и непроверенные данные без статусов;
- есть приватные данные в публичном доступе;
- нет понятного способа предложить исправление.

## Решение о запуске

Решение о запуске принимает менеджер после первого теста и исправления критичных замечаний.
