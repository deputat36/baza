# Ответственные за разделы базы

## Назначение

Матрица ответственности показывает, кто после запуска отвечает за актуальность и качество каждого раздела базы знаний.

Это эксплуатационный слой: он нужен менеджеру, чтобы база не превратилась в устаревший справочник.

## Основной файл

`data/drafts/section-ownership-matrix.csv`

## Отчет

```bash
python scripts/tools/build_section_ownership_report.py
```

Результат:

- `build/section-ownership-report.md`
- `build/section-ownership-report.csv`

## Как использовать

1. Проверить зоны высокой критичности.
2. Назначить реальных ответственных в закрытой рабочей таблице.
3. Указать замещающего на случай отсутствия владельца.
4. Связать каждую зону с отчетом, по которому её можно контролировать.
5. Перевести раздел в `READY`, когда ответственность подтверждена.

## Важно

В публичном репозитории указываются только роли.

Реальные фамилии, телефоны и внутренние контакты сотрудников должны храниться только в закрытой Google Таблице.
