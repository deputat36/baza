# Уровни доверия источников

## Зачем это нужно

В базе будут разные типы данных: официальные сведения, проверенные городские контакты, рабочая практика офиса, черновики, закрытые контакты и архив.

Чтобы сотрудник не принял черновик за подтвержденную информацию, каждой записи нужен понятный уровень доверия.

## Основные уровни

- `OFFICIAL` — данные из официального источника.
- `VERIFIED` — контакт или запись проверены сотрудником офиса.
- `WORKING` — используется в работе, но требует регулярного подтверждения.
- `CHECK_REQUIRED` — нельзя использовать без дополнительной проверки.
- `DRAFT` — черновик, не готовый к рабочему применению.
- `PRIVATE_ONLY` — только для закрытой Google Таблицы.
- `ARCHIVE` — неактуальная запись для истории.

## Главное правило

Если запись не имеет уровня `OFFICIAL`, `VERIFIED` или `WORKING`, ее нельзя использовать как подтвержденную рабочую информацию.

Реальные телефоны, ФИО сотрудников, внутренние условия и данные клиентов не публикуются в репозитории.

