# GO/NO-GO отчет запуска

## Назначение

GO/NO-GO отчет отвечает на один управленческий вопрос: можно ли запускать базу знаний в офисе сейчас.

Отчет собирает блокеры из нескольких источников:

- чек-листа запуска;
- приемочных сценариев;
- плана обучения;
- матрицы владельцев разделов.

## Команды

```bash
make go-no-go
```

Или напрямую:

```bash
python scripts/tools/build_go_no_go_report.py
```

## Результат

- `build/go-no-go-report.md`
- `build/go-no-go-report.csv`

## Правило решения

`GO` возможен только если нет открытых пунктов, которые блокируют запуск, и нет высококритичных зон ответственности без статуса готовности.

Если есть хотя бы один блокер, отчет показывает `NO-GO`.

## Важно

Отчет проверяет управленческую готовность по статусам в CSV.

Он не подтверждает юридическую точность данных, актуальность реальных контактов и качество закрытой Google Таблицы.
