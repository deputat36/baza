# Этап жизненного цикла записей

## Добавлено

- CSV-справочник жизненного цикла записи.
- Генератор отчета по статусам записей.
- Документация по статусам `DRAFT`, `CHECK`, `READY`, `STALE`, `PRIVATE_ONLY`, `BLOCKED`, `ARCHIVE`.
- Цель `make record-lifecycle`.
- Сборка отчета в GitHub Actions и GitHub Pages workflow.
- Подключение к `make preflight` и `make launch-packet`.
- Ежемесячная задача в регламенте эксплуатации.

## Практический смысл

У базы появляется формальное правило: рабочей считается только запись в статусе `READY` и с непросроченной датой следующей проверки. Черновики, закрытые записи, спорные записи и устаревшие данные не должны использоваться как подтвержденная информация.

## Команда

```bash
make record-lifecycle
```

## Результаты

- `build/record-lifecycle-report.md`
- `build/record-lifecycle-report.csv`
