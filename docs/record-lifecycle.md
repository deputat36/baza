# Жизненный цикл записи

## Зачем это нужно

В базе знаний нельзя смешивать черновики, проверенные рабочие записи, устаревшие данные и закрытые контакты.

Жизненный цикл записи задает простое правило: какой статус у записи сейчас, кто отвечает за переход дальше и можно ли использовать запись в работе.

## Статусы

- `DRAFT` — черновик или новая идея.
- `CHECK` — запись заполнена, но требует проверки.
- `READY` — запись проверена и пригодна для работы.
- `STALE` — срок проверки истек.
- `PRIVATE_ONLY` — запись допустима только в закрытой Google Таблице.
- `BLOCKED` — есть риск, спор или юридическая неопределенность.
- `ARCHIVE` — запись больше не актуальна.

## Главное правило

Рабочей считается только запись в статусе `READY` и с непросроченной датой следующей проверки.

Все остальные статусы требуют проверки, доработки, закрытого хранения или архивирования.

## Генератор отчета

```bash
make record-lifecycle
```

Результаты:

- `build/record-lifecycle-report.md`
- `build/record-lifecycle-report.csv`
