# JSON-индекс базы знаний

## Назначение

`knowledge-index.json` — нейтральный read-only экспорт базы знаний из CSV-файлов.

Он нужен, чтобы в будущем можно было подключать базу знаний к другим интерфейсам без немедленного переноса всей структуры в CRM.

## Файл сборки

```text
build/knowledge-index.json
```

## Генератор

```text
scripts/tools/build_knowledge_index.py
```

## Запуск

```bash
python scripts/tools/build_knowledge_index.py
```

Или через общий preflight:

```bash
make preflight
```

## Формат записи

Каждая запись содержит:

- `id` — стабильный ID записи;
- `sheet` — логический раздел;
- `source` — CSV-источник;
- `row_number` — строка в CSV;
- `title` — короткий заголовок;
- `summary` — краткое описание;
- `status` — статус проверки;
- `priority` — приоритет;
- `related` — связанные ID из колонок `Связанные ...`;
- `search_text` — текст для поиска;
- `raw` — исходные непустые поля строки.

## Правило совместимости

Внешние интерфейсы должны использовать стабильные ID и поля верхнего уровня.

Поле `raw` можно использовать для просмотра, но нельзя считать его стабильным API: колонки CSV могут уточняться.

## Ограничение

Этот индекс не должен содержать реальные клиентские данные, приватные телефоны, внутренние условия и закрытые комментарии.

Для боевого использования с реальными данными нужен закрытый источник: закрытая Google Таблица или защищённые таблицы `kb_*` с RLS.
