# Этап контроля актуальности данных

## Добавлено

- Справочник политик актуальности данных.
- Реестр разделов, которые нужно регулярно проверять.
- Скрипт генерации отчёта актуальности.
- Markdown и CSV отчёты актуальности.
- Документация по контролю актуальности.
- Подключение проверки к preflight, GitHub Actions, Pages-сборке, XLSX и индексу артефактов.

## Практический смысл

Менеджер сможет видеть, какие контакты, инструкции и интеграционные правила давно не проверялись.

## Ограничение

Просроченная дата проверки является рабочим сигналом, а не причиной падения CI.
