# Матрица доступа будущей интеграции

## Назначение

Матрица доступа описывает, какие роли смогут читать или изменять разные области данных, если база знаний в будущем будет связана с Навигатором сделок или другим внутренним интерфейсом.

Сейчас это только проектный слой. Он не подключает внешний проект и не меняет Навигатор сделок.

## Файл данных

`data/drafts/integration-access-matrix.csv`

## Что описывается

- аудитория или роль;
- область данных;
- доступ на чтение;
- доступ на изменение;
- место хранения данных;
- причина такого доступа;
- статус проверки.

## Допустимые значения доступа

- `yes` — полный доступ;
- `limited` — ограниченный доступ по роли, сделке или назначению;
- `no` — доступа нет.

## Проверка

```bash
make integration-access
```

или напрямую:

```bash
python scripts/tools/build_integration_access_report.py
```

## Результаты

Скрипт создаёт:

- `build/integration-access-report.md`;
- `build/integration-access-report.csv`.

## Главное правило

Публичный репозиторий может хранить только структуру, правила, обезличенные сценарии и read-only контракты.

Клиентские данные, внутренние комментарии, рабочие телефоны сотрудников и закрытые условия должны оставаться в закрытой Google Таблице или приватной базе данных.
