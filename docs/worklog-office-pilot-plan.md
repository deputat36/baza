# Worklog: office pilot plan

Дата: 09.07.2026

## Задача

Подготовить следующий практический слой проекта после технического `pipeline-health`: маршрут перехода от структуры в GitHub к первому недельному офисному пилоту закрытой базы знаний.

## Почему это нужно

Проект уже содержит много архитектурных документов, генераторов, отчетов и проверок. Следующий риск — база останется набором файлов и не дойдет до ежедневного использования сотрудниками.

Для перехода к рабочему MVP нужен понятный план пилота:

- кто участвует;
- что проверяется;
- какие данные нужны минимально;
- как фиксировать обратную связь;
- какие метрики считать;
- когда принимать решение GO/NO-GO;
- как не нарушить правило публичной безопасности.

## Принцип безопасности

Все материалы остаются безопасными для публичного репозитория.

В документе используются только роли, процессы, шаблоны и правила. Реальные телефоны, ФИО, внутренние контакты, клиентские данные, закрытые условия и комментарии по сделкам не добавлялись.

## Следующий шаг

Перед реальным запуском пилота выполнить:

```bash
make pipeline-health
make preflight
```

Затем проверить, что:

- закрытая Google Таблица создана;
- минимальный набор данных импортирован;
- участникам выдан доступ только к закрытой таблице;
- обратная связь фиксируется в безопасном месте;
- в публичном репозитории нет приватных данных.
