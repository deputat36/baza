# Этап строгой валидации

## Добавлено

- CSV-валидатор теперь возвращает ошибку при структурных проблемах.
- У CSV-валидатора появился режим `--warn-only`.
- Privacy scan получил строгий режим `--strict`.
- Makefile получил команды `validate-warn` и `privacy-strict`.
- Документация уточняет, какие проверки блокируют CI.

## Практический смысл

GitHub Actions теперь может реально остановить повреждённые CSV-файлы, а privacy scan остаётся управляемым предупреждением.

## Следующий шаг

Запустить workflow и исправить CSV-ошибки, если они появятся в логах.
