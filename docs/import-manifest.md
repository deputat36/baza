# Манифест импорта

## Назначение

Файл фиксирует порядок переноса данных из репозитория в рабочую Google Таблицу.

## Базовый порядок

1. Сначала импортировать справочники из `data/dictionaries`.
2. Затем импортировать шаблоны из `templates`.
3. После этого переносить черновики из `data/drafts`.
4. В конце настроить фильтры, статусы и выпадающие списки.

## Минимальный набор для первого теста

Для первого теста достаточно перенести:

- ситуации по недвижимости;
- документы по сделкам;
- государственные органы;
- банки;
- FAQ;
- поисковые синонимы;
- частые ошибки.

## Расширенный набор

После первого теста можно добавить:

- нотариусов;
- кадастровых инженеров;
- оценщиков;
- ЖКХ;
- управляющие компании;
- подрядчиков;
- новостройки;
- обучающие чек-листы;
- пользовательские сценарии.

## После импорта

Нужно проверить:

- уникальность ID;
- статусы;
- категории;
- связанные записи;
- отсутствие приватных данных;
- удобство поиска.

## Главное правило

GitHub хранит структуру и безопасные черновики. Рабочая таблица хранит проверенные данные и реальные контакты в закрытом доступе.
