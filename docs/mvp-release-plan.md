# План выпуска MVP

## Цель MVP

Запустить первую рабочую версию базы знаний в Google Таблице и проверить, помогает ли она сотрудникам быстрее находить ответы.

## Состав MVP

Минимальный набор листов:

1. Главная.
2. Контакты.
3. Ситуации.
4. База знаний.
5. Практика.
6. Предложения изменений.
7. Справочники.

## Что входит в первый выпуск

- Государственные органы.
- Банки.
- Нотариусы как структура.
- Кадастровые инженеры.
- Оценщики.
- ЖКХ.
- Управляющие компании.
- Документы.
- Типовые ситуации.
- FAQ.
- Шаблоны документов.
- Онлайн-сервисы.

## Что не входит в первый выпуск

- Клиентские данные.
- Реальные внутренние телефоны в публичном доступе.
- Коммерчески чувствительные условия.
- Непроверенные личные рекомендации.
- Закрытые материалы компании.

## Этапы запуска

### Этап 1. Подготовка

- Импортировать CSV в Google Таблицу.
- Настроить фильтры.
- Настроить выпадающие списки.
- Скрыть технические поля.
- Назначить ответственных за разделы.

### Этап 2. Проверка

- Проверить первые 30–50 записей.
- Убрать дубли.
- Пометить непроверенное статусом `Нужно проверить`.
- Заполнить реальные контакты только в закрытой таблице.

### Этап 3. Тест

- Дать доступ 2–3 сотрудникам.
- Провести тест по файлу `docs/first-user-test.md`.
- Собрать замечания.

### Этап 4. Доработка

- Исправить структуру.
- Добавить недостающие ситуации.
- Упростить непонятные поля.
- Добавить практические заметки.

### Этап 5. Запуск в офисе

- Дать доступ всем нужным сотрудникам.
- Объяснить правила использования.
- Назначить регулярную проверку.

## Критерии готовности

MVP готов, если:

- сотрудник находит ответ за 1–3 минуты;
- таблица понятна без долгого обучения;
- есть ответственные за разделы;
- есть форма предложений;
- нет персональных данных клиентов;
- реальные контакты хранятся только в закрытом доступе.
