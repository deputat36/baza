# Как посмотреть пользовательский интерфейс

## Что уже можно посмотреть

Сейчас доступен не финальный CRM-интерфейс, а статическое HTML-превью базы знаний и страница артефактов сборки.

Это позволяет проверить:

- структуру разделов;
- поиск по данным;
- навигацию по листам;
- XLSX-сборку;
- отчёты готовности и качества данных;
- планы переноса в закрытую Google Таблицу.

## Вариант 1. GitHub Pages

Это основной способ просмотра в браузере.

1. Открыть репозиторий `deputat36/baza`.
2. Перейти в `Settings -> Pages`.
3. В `Build and deployment` выбрать `Source: GitHub Actions`.
4. Перейти в `Actions -> Publish HTML preview`.
5. Нажать `Run workflow`.
6. Дождаться успешного запуска.
7. Открыть ссылку deployment или адрес Pages.

Ожидаемый адрес:

```text
https://deputat36.github.io/baza/
```

Точный адрес GitHub показывает в настройках Pages и в результате workflow.

## Вариант 2. Артефакт GitHub Actions

Если GitHub Pages ещё не включён, интерфейс можно посмотреть локально.

1. Открыть `Actions -> Validate data files`.
2. Открыть успешный запуск workflow.
3. Скачать артефакт `baza-build-artifacts`.
4. Распаковать архив.
5. Открыть `index.html`.

Основной экран — `index.html`.

Само HTML-превью базы находится здесь:

```text
html-preview/index.html
```

## Что пока не является готовым продуктом

HTML-превью не заменяет закрытую рабочую Google Таблицу.

В нём не должно быть:

- реальных внутренних телефонов;
- персональных данных клиентов;
- закрытых условий компании;
- приватных комментариев по сотрудникам или контрагентам.

Реальные рабочие контакты и проверенные данные нужно переносить только в закрытую Google Таблицу.

## Если workflow падает

Проверить лог упавшего шага.

Частые причины:

- не включён GitHub Pages;
- не прошла проверка CSV;
- не собрался XLSX;
- найден риск публикации приватных данных;
- не прошла smoke-проверка HTML-превью.

После исправления нужно повторно запустить workflow.
