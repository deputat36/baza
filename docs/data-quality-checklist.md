# Чек-лист качества данных

Этот чек-лист используется перед тем, как запись попадает из черновика в рабочую Google Таблицу или будущую CRM.

## Проверка контакта

- У записи есть ID.
- Указан тип объекта.
- Указана категория и подкатегория.
- Название написано понятно для сотрудника.
- Телефон и адрес не добавлены без проверки.
- Если контакт внутренний или персональный, он не публикуется в открытом репозитории.
- Есть статус записи.
- Есть источник информации.
- Есть дата последней проверки или пометка `Нужно проверить`.

## Проверка инструкции

- Инструкция отвечает на конкретный рабочий вопрос.
- Есть пошаговый порядок действий.
- Указано, какие документы нужны.
- Указано, куда обращаться.
- Указаны частые ошибки.
- Указаны связанные контакты и документы.
- Нет спорных юридических утверждений без проверки юристом.

## Проверка ситуации

- Ситуация сформулирована языком сотрудника.
- Есть короткий ответ, что делать.
- Есть список рисков.
- Есть список частых ошибок.
- Указано, когда нужно подключать юриста, ипотечного брокера или менеджера.
- Указаны связанные документы и организации.

## Проверка практической заметки

- Заметка короткая и применимая.
- Понятно, к какой записи относится опыт.
- Указан автор.
- Указана дата.
- Заметка не содержит персональные данные клиента.
- Заметка не является слухом или непроверенным обвинением.

## Минимальные требования для публикации

Запись можно считать готовой к использованию, если заполнены:

- ID;
- тип;
- категория;
- название;
- когда использовать;
- статус;
- источник;
- приоритет.

## Статус `Проверено`

Статус `Проверено` ставится только если:

- информация проверена по официальному источнику или ответственным сотрудником;
- нет явных дублей;
- нет персональных данных клиентов;
- запись понятна новому сотруднику без дополнительных объяснений.
