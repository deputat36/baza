# Правила заполнения базы знаний

## 1. Не добавлять лишнее

Каждая запись должна помогать сотруднику быстрее и качественнее провести сделку.

Если информация не помогает работе специалиста, юриста, ипотечного брокера или менеджера — её не нужно добавлять в основную базу.

## 2. Разделять факты и практику

Факт:

- официальный телефон;
- адрес;
- график;
- перечень услуг;
- ссылка на официальный сайт.

Практика:

- быстрее отвечают по почте;
- лучше обращаться утром;
- по пятницам бывают очереди;
- конкретный специалист хорошо решает сложные вопросы.

Факты должны проверяться по официальным источникам. Практика должна подписываться автором заметки.

## 3. Не хранить персональные данные клиентов

Запрещено добавлять:

- ФИО клиентов;
- телефоны клиентов;
- адреса объектов клиентов без необходимости;
- суммы сделок конкретных клиентов;
- паспортные данные;
- сканы документов;
- переписку.

## 4. Внутренние контакты хранить осторожно

Если в базе появляются личные телефоны сотрудников, кураторов, представителей ЦО или партнёров, репозиторий должен быть приватным.

В публичный репозиторий можно добавлять только структуру, шаблоны и обезличенные примеры.

## 5. У каждой записи должен быть статус

Допустимые статусы:

- Проверено.
- Нужно проверить.
- Устарело.
- Не работает.
- Не рекомендовать.

## 6. Указывать дату проверки

Любой контакт без даты проверки быстро теряет ценность.

Рекомендуемый формат даты: `ДД.ММ.ГГГГ`.

## 7. Указывать источник

Источник может быть:

- официальный сайт;
- Google Диск офиса;
- сотрудник офиса;
- звонок в организацию;
- личный опыт сделки;
- открытый справочник.

## 8. Практические заметки оформлять коротко

Плохой вариант:

«В МФЦ бывают очереди, иногда долго».

Хороший вариант:

«По опыту сотрудников, во вторник после 15:00 очередь меньше. Проверить актуальность перед использованием».

## 9. Не удалять спорные записи сразу

Если контакт сомнительный, его лучше пометить статусом `Нужно проверить` или `Не рекомендовать`, а не удалять. Это сохранит историю опыта.

## 10. Новые предложения сотрудников

Для предложений сотрудников используется простой формат:

| Поле | Описание |
|---|---|
| Что изменить | Краткое описание |
| Что было | Старое значение |
| Что стало | Новое значение |
| Почему важно | Как влияет на работу |
| Автор | Кто предложил |
| Дата | Дата предложения |
| Статус | На проверке, принято, отклонено |
