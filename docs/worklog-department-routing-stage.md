# Этап маршрутизации в департаменты центрального офиса

## Добавлено

- Черновик маршрутизации по департаментам центрального офиса без реальных контактов.
- Генератор отчета по маршрутизации.
- Документация по заполнению закрытой таблицы с кураторами.
- Цель `make department-routing`.
- Сборка отчета в GitHub Actions и GitHub Pages workflow.
- Отчет в пакете запуска, preflight-сводке и стартовом индексе артефактов.
- Ежемесячная задача в регламенте эксплуатации.

## Практический смысл

Менеджер получает структуру, куда направлять юридические, ипотечные, IT, маркетинговые, финансовые, HR и другие вопросы в центральный офис.

## Безопасность

Реальные ФИО, телефоны, почты, чаты и внутренние каналы не добавляются в публичный репозиторий. Они должны заполняться только в закрытой Google Таблице.

## Команда

```bash
make department-routing
```

## Результаты

- `build/department-routing-report.md`
- `build/department-routing-report.csv`
