# Структура базы знаний

## Основная идея

База знаний должна хранить не только контакты, но и связанные рабочие объекты: документы, инструкции, ситуации, организации, подрядчиков, банки, шаблоны и практические заметки сотрудников.

## Основные сущности

| Код | Тип объекта | Пример |
|---:|---|---|
| ORG | Организация | МФЦ, Росреестр, Администрация |
| CNT | Контакт | Специалист, менеджер банка, помощник нотариуса |
| DOC | Документ | Выписка ЕГРН, согласие супруга |
| INS | Инструкция | Как получить выписку ЕГРН |
| ALG | Алгоритм | Продажа квартиры с ипотекой |
| TMP | Шаблон | Расписка, авансовое соглашение |
| SIT | Ситуация | Продажа квартиры с детскими долями |
| SRV | Сервис | Госуслуги, ФНС, ФССП |
| PRC | Практика | Локальный совет сотрудника |
| DEV | Новостройка | Новый жилой комплекс |
| BNK | Банк | Сбер, ВТБ, Дом.РФ |
| PTR | Партнёр | Оценщик, кадастровый инженер, подрядчик |

## Обязательные поля основной таблицы

| Поле | Назначение |
|---|---|
| ID | Уникальный код записи |
| Тип | Тип объекта: организация, контакт, документ, инструкция |
| Категория | Крупный раздел базы |
| Подкатегория | Уточняющий раздел |
| Название | Название организации, документа или инструкции |
| Контактное лицо | ФИО, если применимо |
| Должность | Должность контактного лица |
| Телефон | Основной телефон |
| Дополнительный телефон | Резервный телефон |
| Email | Электронная почта |
| Адрес | Физический адрес |
| Сайт | Официальная ссылка |
| Когда обращаться | В каких ситуациях нужна запись |
| Документы | Какие документы могут потребоваться |
| Лайфхак | Практический совет |
| Частые ошибки | Что сотрудники часто делают неправильно |
| Важно | Критичное предупреждение |
| Связанные записи | ID связанных объектов |
| Приоритет | Значимость записи |
| Статус | Проверено, нужно проверить, устарело |
| Последняя проверка | Дата проверки |
| Проверил | Кто проверил |

## Приоритеты

| Приоритет | Значение |
|---|---|
| 5 | Критически важно, используется часто |
| 4 | Используется регулярно |
| 3 | Используется иногда |
| 2 | Редко |
| 1 | Справочная информация |

## Статусы

- Проверено.
- Нужно проверить.
- Устарело.
- Изменился телефон.
- Изменился адрес.
- Не рекомендовать.

## Связи между объектами

Пример:

- `SIT-0001 Продажа квартиры с детскими долями` связана с `ORG-0305 Опека`, `DOC-0010 Разрешение опеки`, `TMP-0022 Заявление`, `INS-0015 Порядок обращения в опеку`.
- `DOC-0001 Выписка ЕГРН` связана с `ORG-0301 Росреестр`, `ORG-0302 МФЦ`, `SRV-0001 Госуслуги`.
