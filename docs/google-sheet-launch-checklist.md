# Чек-лист запуска Google Таблицы

## До импорта

- Создать закрытую Google Таблицу.
- Назначить владельца.
- Собрать планы настройки: `make tabs-plan`, `make validation-plan`, `make formatting-plan`.
- Создать листы по `build/google-sheet-tabs-plan.md` или через `setupBazaKnowledgeSheet`.
- Подготовить справочники статусов, категорий, типов и приоритетов.
- Проверить, что в импортируемых файлах нет приватных данных.

## Импорт данных

Импортировать файлы в порядке из:

```text
build/import-plan.md
```

Базовый порядок:

1. Справочники.
2. Контакты.
3. Ситуации.
4. База знаний.
5. Документы.
6. FAQ.
7. Подрядчики.
8. Новостройки.
9. Практика.
10. Предложения изменений.

## Настройка таблицы

- Закрепить первую строку на каждом рабочем листе.
- Включить фильтры.
- Настроить ширину колонок.
- Скрыть технические поля для сотрудников.
- Добавить выпадающие списки для статусов, категорий и приоритетов.
- Добавить условное форматирование по статусам и приоритетам.
- Защитить справочники и системные поля.

Для ускорения можно использовать:

```text
scripts/google-apps-script/setup-baza.gs
scripts/google-apps-script/setup_baza_sheet.gs
scripts/google-apps-script/menu.gs
```

Ручной порядок функций:

```text
setupBazaKnowledgeSheet
applyBazaSheetFormattingPlan
```

Через меню Google Таблицы:

```text
База знаний -> Полная подготовка таблицы
```

## Проверка перед выдачей сотрудникам

- Запустить `validateBazaKnowledgeSheet` из `scripts/google-apps-script/validate-baza-sheet.gs` или пункт `База знаний -> 3. Проверить данные`.
- Открыть лист `Проверка данных`.
- Исправить пустые ID, дубли заголовков, пустые заголовки, неверные статусы и приоритеты.
- Запустить проверку повторно.
- Проверить первые 30 записей.
- Убедиться, что поиск работает по основным словам.
- Проверить связи по ID.
- Убедиться, что непроверенные записи имеют статус `CHECK` или `DRAFT`.
- Убедиться, что реальные контакты заполнены только в закрытой таблице.

## Первый показ сотрудникам

На планёрке показать 3 сценария:

1. Как найти ситуацию.
2. Как найти контакт.
3. Как предложить правку.

Не нужно проводить длинное обучение. Достаточно показать пользу на реальных рабочих вопросах.

## Первый день использования

Попросить сотрудников искать ответы через таблицу по любым типовым вопросам.

Менеджеру важно фиксировать:

- что не нашли;
- что нашли слишком долго;
- какие слова вводили в поиск;
- какие разделы непонятны;
- какие поля лишние.

## Через неделю

Провести короткий разбор:

- что помогло;
- что мешало;
- какие разделы нужны срочно;
- какие записи нужно проверить;
- кто готов помогать с наполнением.
