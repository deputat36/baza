# Обучение сотрудников и работа с базой знаний

## Назначение

Проект содержит два связанных, но разных слоя обучения:

1. Профессиональное обучение специалиста по недвижимости.
2. Обучение сотрудников работе с базой знаний.

Их нельзя смешивать.

Обучение работе с базой отвечает на вопрос «где найти проверенную информацию». Профессиональная программа отвечает на вопрос «что сотрудник должен уметь делать в работе».

## Архитектура профессионального обучения

Основной документ:

- `docs/training-system-architecture.md`.

Справочники:

- `data/dictionaries/training-stages.csv` — этапы от кандидата до развития СПН 6+ и наставничества;
- `data/dictionaries/training-competencies.csv` — единый словарь профессиональных компетенций.

Рабочие структуры:

- `data/drafts/training-learning-paths.csv` — ролевые маршруты;
- `data/drafts/training-modules.csv` — практические учебные модули;
- `data/drafts/training-assessments.csv` — контрольные точки и критерии перехода;
- `data/drafts/training-materials-registry.csv` — безопасный реестр мастер-материалов.

## Архитектура обучения работе с базой

Основной файл:

- `data/drafts/role-training-plan.csv`.

Он проверяет, что менеджер, СПН, юрист, ипотечный специалист и администратор умеют:

- найти свой маршрут;
- открыть нужный отчет или раздел;
- отличить черновик от проверенной записи;
- предложить исправление;
- соблюдать правила публичной безопасности.

## Единый операционный принцип

`Космос` является источником фактических рабочих действий.

В учебной системе не нужно повторно записывать количество звонков, встреч, подборок, показов и сделок, если эти данные уже отражены в CRM.

Для подтверждения модуля достаточно:

- действия или результата в `Космосе`;
- наблюдения проверяющего;
- одного краткого вывода по качеству;
- одного следующего шага.

## Отчет по обучению работе с базой

```bash
python scripts/tools/build_role_training_report.py
```

Результат:

- `build/role-training-report.md`;
- `build/role-training-report.csv`.

Этот отчет не является полной картой профессиональных компетенций. Он остается приемочным отчетом для запуска базы знаний.

## Как использовать объединенную систему

1. Определить роль и текущий этап сотрудника.
2. Открыть соответствующий маршрут в `training-learning-paths.csv`.
3. Выбрать обязательные модули из `training-modules.csv`.
4. Открыть утвержденный мастер-материал в закрытом Drive.
5. Выполнить практическое действие.
6. Проверить качество и при необходимости факт в `Космосе`.
7. Зафиксировать статус и один следующий шаг в закрытой учебной таблице.
8. На контрольной точке принять решение о переходе на следующий этап.

## Критерий прохождения

Модуль не считается пройденным после чтения файла или формальной подписи.

Сотрудник должен:

1. Объяснить порядок своими словами.
2. Выполнить практику.
3. Показать результат в `Космосе`, если модуль связан с операционной работой.
4. Не допустить критической ошибки.
5. Понимать следующий шаг.

## EVP и УТП

В обучении используются отдельно:

- EVP — для найма, адаптации, развития и удержания сотрудников;
- УТП — для клиентской презентации;
- ценность риелтора — для объяснения личного вклада специалиста в результат клиента.

Цифры, гарантии, скидки, рейтинги, количество городов и другие быстро меняющиеся сведения нельзя использовать без локальной проверки, источника и даты актуальности.

## Безопасность и права использования

В публичном репозитории хранится только структура обучения.

ФИО сотрудников, результаты прохождения, ссылки на доказательства, внутренние комментарии, закрытые условия и мастер-материалы должны храниться в закрытом контуре.

Материалы с ограничениями на копирование нельзя переносить целиком в GitHub. Для них в публичном реестре хранится только безопасная метаинформация.
