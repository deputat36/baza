# Проверка актуальности контактов

## Назначение

Регламент нужен, чтобы закрытая Google Таблица с контактами оставалась рабочей: телефоны, графики, ответственные и каналы связи должны регулярно проверяться.

Публичный репозиторий хранит только процесс проверки, а не реальные контакты.

## Исходный файл

`data/drafts/contact-verification-workflow.csv`

## Генератор

`scripts/tools/build_contact_verification_report.py`

## Результаты

- `build/contact-verification-report.md`
- `build/contact-verification-report.csv`

## Запуск

```bash
make contact-verification
```

Или в составе полной проверки:

```bash
make preflight
```

## Что фиксировать в закрытой таблице

- Дата проверки.
- Кто проверил.
- Что именно проверено.
- Источник проверки.
- Новый статус контакта.
- Следующая дата проверки.
- Комментарий для внутреннего использования.

## Что нельзя публиковать

- ФИО и телефоны непубличных контактов.
- Внутренние чаты и почты.
- Комментарии по людям и подрядчикам.
- Данные клиентов.
- Закрытые условия взаимодействия.
