# План тестового MVP v0.3

## Что означает v0.3

`v0.3` — это этап, когда закрытая Google Таблица уже собрана и передана небольшой группе сотрудников для проверки на реальных рабочих вопросах.

## Цель теста

Понять, помогает ли база находить ответы быстрее, чем переписка в чатах или обращение к менеджеру.

## Кто участвует

Минимальная группа:

- один новичок;
- один опытный специалист;
- менеджер или наставник.

По возможности подключить:

- юриста;
- ипотечного брокера.

## Что тестируем

Тестируем не полноту базы, а удобство использования.

Нужно проверить:

- понятность листов;
- скорость поиска;
- достаточность описаний;
- полезность связанных записей;
- понятность статусов;
- удобство предложения правок.

## Задания

1. Найти порядок действий по объекту с детскими долями.
2. Найти, что делать при утере документа.
3. Найти документ для первичной проверки объекта.
4. Найти, кого подключить по ипотечной сделке.
5. Найти, как добавить правку или полезный совет.

## Что фиксировать

Во время теста фиксировать:

- какой запрос вводил сотрудник;
- нашёл ли он ответ;
- сколько времени занял поиск;
- что было непонятно;
- какие слова нужно добавить в синонимы;
- какие поля лишние.

## Итог теста

После теста нужно получить:

- список ошибок;
- список лишних полей;
- список недостающих ситуаций;
- список новых синонимов;
- список контактов, которые нужно проверить;
- решение о запуске на весь офис или повторном тесте.

## Критерий успеха

Тест успешен, если большинство типовых ответов находится за 1–3 минуты, а сотрудники понимают, куда вносить правки.
