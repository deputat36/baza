# Безопасность и приватность

## Главное правило

Публичный репозиторий не должен содержать внутренние контакты, персональные данные клиентов и коммерчески чувствительную информацию.

## Нельзя хранить в публичном репозитории

- ФИО клиентов.
- Телефоны клиентов.
- Паспортные данные.
- Адреса объектов клиентов без обезличивания.
- Суммы и условия конкретных сделок.
- Сканы документов.
- Внутренние телефоны сотрудников и кураторов.
- Личные телефоны партнёров без разрешения.
- Пароли, токены, ключи API.
- Внутренние коммерческие условия.

## Что можно хранить

- Структуру базы знаний.
- Обезличенные шаблоны.
- Общие инструкции.
- Публичные контакты организаций после проверки.
- Черновики без личных данных.
- Описание будущего CRM-модуля.

## Где хранить рабочие контакты

Рабочие телефоны сотрудников, партнёров, кураторов и представителей центрального офиса лучше хранить в приватной Google Таблице или приватном репозитории.

## Статусы безопасности

- `PUBLIC` — можно хранить публично.
- `PRIVATE` — только для внутреннего доступа.
- `CHECK` — требуется проверка перед публикацией.
- `REMOVE` — удалить из публичного доступа.

## Перед публикацией проверить

1. Нет ли персональных данных клиента.
2. Нет ли внутренних телефонов без необходимости.
3. Нет ли коммерческой информации.
4. Нет ли непроверенных обвинений или эмоциональных отзывов.
5. Есть ли источник.
6. Есть ли статус.

## Рекомендация

Перед реальным наполнением контактами сделать репозиторий приватным или хранить фактические контакты только в Google Таблице с ограниченным доступом.
