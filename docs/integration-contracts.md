# Контракты будущей интеграции

## Назначение

Контракты интеграции описывают, какие файлы проекта можно рассматривать как стабильные read-only точки подключения для будущего интерфейса.

Проект Навигатора сделок сейчас не меняется.

## Источник

```text
data/dictionaries/integration-contracts.csv
```

## Что считается контрактом

Контрактом считается связка:

- источник данных;
- артефакт сборки;
- документация;
- генератор;
- версия;
- статус готовности.

## Проверка

```bash
make integration-contracts
```

Полная проверка:

```bash
make preflight
```

## Практический смысл

Если в будущем база знаний будет подключаться к другому интерфейсу, разработчику не нужно будет угадывать, какие файлы использовать.

Нужно смотреть на реестр контрактов и отчёт сборки.
