# План сборки рабочей Google Таблицы

## Цель

Собрать из репозитория первую рабочую таблицу, которой сможет пользоваться сотрудник офиса.

## Что берём из GitHub

Из папки `templates/`:

- `contacts-template.csv`;
- `knowledge-template.csv`;
- `situations-template.csv`;
- `change-request-template.csv`;
- `practice-note-template.csv`.

Из папки `data/drafts/`:

- государственные органы;
- документы;
- ситуации;
- банки;
- нотариусы;
- кадастровые инженеры;
- оценщики;
- ЖКХ;
- УК;
- подрядчики;
- FAQ;
- шаблоны документов;
- новостройки;
- обучающие чек-листы.

Из папки `data/dictionaries/`:

- категории;
- статусы;
- приоритеты;
- типы объектов;
- роли;
- ключевые слова;
- базовые поля.

## Порядок сборки

1. Создать приватную Google Таблицу.
2. Создать листы по макету.
3. Импортировать справочники.
4. Импортировать основные данные.
5. Настроить фильтры.
6. Закрепить первую строку.
7. Скрыть технические поля.
8. Настроить выпадающие списки.
9. Добавить быстрые ссылки на главную.
10. Дать доступ тестовой группе.

## Листы первого запуска

- Главная.
- Контакты.
- Ситуации.
- База знаний.
- Практика.
- Предложения изменений.
- Справочники.
- Ключевые слова.
- Партнёры и оценка.

## Что заполняется вручную

Вручную в закрытой таблице заполняются:

- реальные телефоны;
- реальные email;
- адреса;
- графики работы;
- ответственные сотрудники;
- практические комментарии;
- проверенные партнёры.

## Что не переносим в открытую версию

- контакты клиентов;
- документы клиентов;
- внутренние условия;
- закрытые телефоны;
- личные комментарии по сделкам.

## Готовность к тесту

Таблица готова к первому тесту, когда сотрудник может найти минимум 10 типовых ситуаций через обычный поиск.
