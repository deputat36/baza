# Проверка данных через GitHub Actions

Workflow находится здесь:

`.github/workflows/validate-data.yml`

Он запускает:

- проверку CSV-файлов;
- проверку обязательных источников XLSX;
- предупреждающий privacy scan;
- сборку промежуточного XLSX;
- технический отчёт по CSV;
- инвентаризацию файлов.

Проверка выполняется при изменениях в ветке `main` и может запускаться вручную.

## Что блокирует workflow

Workflow упадёт, если:

- CSV-файлы имеют структурные ошибки;
- обязательный источник XLSX отсутствует или не читается;
- XLSX-файл не собирается;
- один из Python-скриптов завершился с ошибкой.

Privacy scan в workflow работает как предупреждающий отчёт. Он показывает потенциальные риски, но не блокирует сборку.

## Что делать при ошибке

Если проверка падает, нужно открыть лог, найти проблемный CSV или скрипт, исправить данные и повторить запуск.

Важно: эта проверка техническая. Она не подтверждает актуальность контактов и юридическую точность данных.
