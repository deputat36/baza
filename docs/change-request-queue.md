# Очередь предложений и исправлений

## Назначение

Очередь изменений нужна, чтобы сотрудники могли предлагать улучшения базы знаний, а менеджер видел это как управляемый список задач.

Через очередь можно фиксировать:

- новый полезный контакт;
- исправление телефона, графика или адреса;
- уточнение инструкции;
- проблему в подсказке;
- предложение по будущей интеграции;
- замечание после пользовательского теста.

## Файлы

- `data/dictionaries/change-request-statuses.csv` — статусы заявок.
- `data/drafts/change-request-queue.csv` — рабочая очередь заявок.
- `templates/change-request-template.csv` — базовый шаблон заявки.

## Проверка

```bash
make change-requests
```

или напрямую:

```bash
python scripts/tools/build_change_request_report.py
```

## Результаты

Скрипт создаёт:

- `build/change-request-report.md`;
- `build/change-request-report.csv`.

## Принцип работы

1. Сотрудник формулирует, что нужно изменить.
2. Менеджер или ответственный переводит заявку в `TRIAGE` или `CHECK`.
3. После проверки заявка получает `APPROVED`, `APPLIED`, `REJECTED` или другой финальный статус.
4. В публичный репозиторий не вносятся приватные телефоны, клиентские данные и закрытые условия.

## Важно

Если заявка требует приватных данных, ей нужно поставить статус `NEEDS_PRIVATE_DATA` и переносить работу в закрытую Google Таблицу или приватный контур.
