# Реестр управленческих действий

## Назначение

Реестр управленческих действий нужен, чтобы менеджер видел не только отчеты, но и конкретный список задач, которые нужно разобрать.

Он собирает действия из:

- еженедельного дайджеста менеджера;
- плана улучшений после запуска;
- очереди предложений сотрудников;
- отчета актуальности данных;
- GO/NO-GO отчета.

## Генератор

`scripts/tools/build_manager_action_register.py`

## Результаты

После запуска появляются файлы:

- `build/manager-action-register.md` — список действий для разбора;
- `build/manager-action-register.csv` — табличная версия реестра.

## Запуск

```bash
make manager-actions
```

Или в составе полной проверки:

```bash
make preflight
```

## Как использовать

1. Открыть `build/manager-action-register.md`.
2. Посмотреть открытые действия и высокий приоритет.
3. Назначить ответственных и сроки в закрытой рабочей таблице или офисном канале.
4. После исправления обновить исходные данные и повторить сборку.

## Ограничение

Реестр не должен содержать реальные внутренние телефоны, данные клиентов и закрытые комментарии по сделкам.

Если действие требует приватной информации, она фиксируется только в закрытом рабочем контуре.
