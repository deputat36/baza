# Этап плана улучшений после запуска

## Добавлено

- CSV-шаблон плана улучшений.
- Генератор плана улучшений после запуска.
- Документация по плану улучшений.
- Команда `make adoption-plan`.
- Сборка плана в `make preflight`.
- Сборка плана в GitHub Actions и Pages preview.
- План добавлен в пакет запуска, preflight-сводку, HTML-индекс артефактов и README.
- После запуска офисный чек-лист ссылается на `build/adoption-improvement-plan.md`.

## Практический смысл

Метрики использования превращаются в конкретные действия: обучение, доработка поиска, добавление синонимов, разбор предложений и назначение ответственных.

## Следующий шаг

Собрать план через `make adoption-plan` или `make preflight`, проверить автоматические сигналы вручную и перенести подтвержденные действия в закрытую рабочую таблицу.
