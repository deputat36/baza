# Проверка закрытой Google Таблицы через Apps Script

## Назначение

Файл `scripts/google-apps-script/validate-baza-sheet.gs` проверяет уже собранную закрытую Google Таблицу базы знаний.

Скрипт работает только внутри активной таблицы и не отправляет данные наружу.

## Запускаемая функция

```text
validateBazaKnowledgeSheet
```

## Что проверяется

- Пустые листы.
- Листы без строк данных.
- Пустые заголовки.
- Дубли заголовков.
- Пустой ID, если в листе есть колонка `ID`.
- Недопустимые значения в колонке `Статус`.
- Недопустимые значения в колонке `Приоритет`.

## Результат

После запуска создаётся или обновляется лист:

```text
Проверка данных
```

В нём будут строки с замечаниями:

```text
Дата проверки | Лист | Строка | Колонка | Проблема | Значение или комментарий
```

Если замечаний нет, скрипт добавит строку `Замечаний не найдено`.

## Когда запускать

- После первого импорта CSV.
- После массового редактирования данных.
- Перед тестом на сотрудниках.
- Перед запуском таблицы на весь офис.
- После добавления нового листа или нового раздела.

## Что делать после проверки

1. Открыть лист `Проверка данных`.
2. Исправить замечания в рабочих листах.
3. Запустить `validateBazaKnowledgeSheet` повторно.
4. Убедиться, что критичных замечаний не осталось.

## Ограничения

Скрипт не проверяет актуальность телефонов, юридическую точность и полноту информации.

Это техническая проверка структуры и базовых значений.
