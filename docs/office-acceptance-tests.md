# Приемочные сценарии офиса

## Назначение

Приемочные сценарии проверяют, сможет ли сотрудник реально пользоваться базой знаний перед запуском в офисе.

Это не техническая проверка CSV. Это список рабочих действий, которые должны пройти менеджер, СПН, юрист, брокер и администратор базы.

## Основной файл

`data/drafts/office-acceptance-tests.csv`

## Отчет

```bash
python scripts/tools/build_office_acceptance_test_report.py
```

Результат:

- `build/office-acceptance-test-report.md`
- `build/office-acceptance-test-report.csv`

## Как использовать

1. Открыть отчет.
2. Пройти сценарии по ролям.
3. Зафиксировать, где сотрудник не понял следующий шаг.
4. Перенести реальные результаты теста в закрытую рабочую таблицу.
5. Блокирующие сценарии перевести в `READY` или `DONE` только после ручной проверки.

## Важно

Сценарии не должны содержать реальные сделки, телефоны клиентов, внутренние комментарии и закрытые условия.
