# Журнал решения запуска

## Назначение

Журнал решения запуска фиксирует управленческое решение после проверки GO/NO-GO.

Он отвечает на вопросы:

- кто принял решение;
- какая версия запускается;
- какое решение принято: `GO`, `NO-GO`, `POSTPONED` или `PENDING`;
- на каком основании принято решение;
- какие условия или блокеры остались;
- когда нужна следующая проверка.

## Команды

```bash
make launch-decision
```

Или напрямую:

```bash
python scripts/tools/build_launch_decision_log.py
```

## Результат

- `build/launch-decision-log.md`
- `build/launch-decision-log.csv`

## Источник

- `data/drafts/launch-decision-log.csv`

## Правило

Решение считается финальным только если поле `Решение` равно `GO`, `NO-GO` или `POSTPONED`, а статус записи закрыт как готовый или утвержденный.

До этого решение считается черновиком и не должно использоваться как разрешение на запуск.
