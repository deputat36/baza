# План улучшений после запуска

## Назначение

План улучшений после запуска нужен, чтобы база знаний не остановилась на этапе публикации.

Он собирает:

- ручные задачи менеджера;
- сигналы из метрик использования;
- проблемы поиска;
- признаки слабого внедрения у отдельных ролей;
- действия по доработке с ответственными.

## Команды

```bash
make adoption-plan
```

Или напрямую:

```bash
python scripts/tools/build_adoption_improvement_plan.py
```

## Результат

- `build/adoption-improvement-plan.md`
- `build/adoption-improvement-plan.csv`

## Источники

- `data/drafts/adoption-improvement-plan.csv`
- `build/usage-metrics-report.csv`

## Правило работы

Сначала разбираются действия с высоким приоритетом.

Автоматические действия из метрик нужно проверить вручную перед переносом в закрытую рабочую таблицу.
