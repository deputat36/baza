# Отчёт по связям базы знаний

## Назначение

`relationship-report.md` показывает, насколько записи базы знаний связаны между собой.

Это важно для будущей интеграции с карточкой сделки: интерфейс сможет показывать релевантные подсказки только если у записей есть стабильные ID и корректные связи.

## Генератор

```text
scripts/tools/build_relationship_report.py
```

## Артефакты

```text
build/relationship-report.md
build/relationship-report.csv
```

## Запуск

```bash
python scripts/tools/build_knowledge_index.py
python scripts/tools/validate_knowledge_index.py
python scripts/tools/build_relationship_report.py
```

Или через общий набор проверок:

```bash
make preflight
```

## Что смотреть в отчёте

- отсутствующие связанные ID;
- записи без входящих и исходящих связей;
- количество связей между `SIT-*`, `DOC-*`, `ORG-*`, `INS-*` и другими ID.

## Практический смысл

Если у ситуации есть связанные документы и контакты, будущая карточка сделки сможет показать сотруднику не просто текст ситуации, а полный рабочий набор:

- что проверить;
- какие документы нужны;
- куда обратиться;
- какие ошибки бывают;
- кто обычно проверяет.
