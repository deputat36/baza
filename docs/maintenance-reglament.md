# Регламент обслуживания базы знаний

## Цель

Поддерживать базу в актуальном и понятном состоянии.

## Роли

- администратор базы;
- ответственный за раздел;
- эксперт для проверки.

## Раз в неделю

1. Смотреть новые предложения изменений.
2. Проверять практические заметки.
3. Исправлять явные ошибки.
4. Убирать дубли.
5. Помечать спорные записи статусом `Нужно проверить`.

## Раз в месяц

1. Проверять записи с приоритетом 5.
2. Обновлять банковский блок.
3. Проверять основные городские контакты.
4. Проверять партнёров.
5. Собирать обратную связь от сотрудников.

## Раз в квартал

1. Пересматривать структуру.
2. Убирать лишние поля.
3. Добавлять новые ситуации.
4. Проверять права доступа.
5. Делать резервную копию рабочей таблицы.

## Статусы

- `Проверено` — можно использовать.
- `Нужно проверить` — требуется уточнение.
- `Черновик` — запись не готова.
- `Устарело` — запись не актуальна.
- `Не рекомендовать` — не использовать в работе.

## Хорошая запись

Хорошая запись имеет:

- понятное название;
- категорию;
- тип;
- рабочее описание;
- приоритет;
- статус;
- дату проверки.

## Важное правило

Реальные внутренние контакты и рабочие комментарии хранятся только в приватной таблице.
