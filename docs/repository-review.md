# Обзор состояния репозитория

## Текущее состояние

Репозиторий содержит безопасный прототип базы знаний для специалиста по недвижимости.

На текущем этапе это не готовая рабочая база с реальными контактами, а структура, шаблоны и черновики для дальнейшего переноса в закрытую Google Таблицу.

## Сильные стороны

- Единая логика ID.
- Разделение контактов, ситуаций, документов и практики.
- Есть статусы проверки.
- Есть защита от публикации приватных данных.
- Структура подходит для переноса в Google Таблицу.
- Проект можно использовать для адаптации новичков.

## Что требует доработки

- Проверить единый формат колонок в CSV.
- Убрать дубли между похожими разделами.
- Добавить больше синонимов для поиска.
- Заполнить реальные контакты в закрытой таблице.
- Назначить ответственных за разделы.
- Провести первый пользовательский тест.
- Добавить локальную практику Борисоглебска.

## Риски

### Риск 1. База станет слишком сложной

Решение: скрывать технические поля и оставлять сотруднику простой интерфейс.

### Риск 2. Данные устареют

Решение: у каждой важной записи должен быть ответственный и дата проверки.

### Риск 3. В публичный репозиторий попадут приватные данные

Решение: хранить реальные контакты только в закрытой Google Таблице и проверять файлы перед публикацией.

### Риск 4. Сотрудники не будут пользоваться базой

Решение: запускать через реальные рабочие ситуации, а не через длинную презентацию.

## Следующий лучший шаг

Создать закрытую Google Таблицу и импортировать туда текущие CSV.

После импорта нужно провести первый тест на 2–3 сотрудниках и исправить структуру по фактическому использованию.

## Рекомендация по дальнейшей разработке

Пока проект развивается как методическая база и набор шаблонов, публичный репозиторий допустим.

Как только появятся реальные внутренние контакты, прямые телефоны партнёров, закрытые условия или комментарии по людям, эти данные нужно хранить только в закрытой таблице либо перевести репозиторий в приватный режим.
