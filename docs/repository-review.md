# Обзор состояния репозитория

## Текущий статус

Репозиторий содержит архитектуру, правила, шаблоны и черновики данных для базы знаний.

Проект уже можно использовать как основу для первой Google Таблицы, но его нельзя считать финальной рабочей базой до проверки и заполнения реальных контактов в закрытом доступе.

## Сильные стороны

- Есть понятная структура.
- Есть разделение на контакты, документы, ситуации, FAQ и практику.
- Есть правила безопасности.
- Есть сценарий первого теста.
- Есть шаблоны для предложений сотрудников.
- Есть черновики большинства ключевых разделов.

## Что требует проверки

- Актуальность городских контактов.
- Реальные телефоны организаций.
- Список управляющих компаний.
- Нотариусы и их специализация.
- Кадастровые инженеры и оценщики.
- Банковские требования и ипотечные программы.
- Полезность полей после теста сотрудниками.

## Что нельзя добавлять в публичный репозиторий

- Персональные данные клиентов.
- Внутренние телефоны сотрудников.
- Личные контакты партнёров без разрешения.
- Коммерческие условия.
- Закрытые документы компании.

## Ближайшие действия

1. Импортировать черновики в приватную Google Таблицу.
2. Настроить листы и фильтры.
3. Заполнить реальные контакты в закрытом доступе.
4. Провести первый тест сотрудниками.
5. Упростить структуру по итогам теста.
6. Назначить ответственных за разделы.

## Рекомендация

До появления реальных контактов можно продолжать работу в публичном репозитории. Как только начнут добавляться внутренние данные, нужно сделать репозиторий приватным или хранить такие данные только в закрытой таблице.
