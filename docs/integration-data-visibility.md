# Видимость данных будущей интеграции

## Назначение

Слой видимости данных фиксирует, какие данные можно хранить в публичном репозитории, а какие должны оставаться только в закрытой Google Таблице или приватной базе.

Это особенно важно перед будущей связкой с другим интерфейсом.

## Источник

```text
data/dictionaries/integration-data-visibility.csv
```

## Команда

```bash
make integration-visibility
```

Полная проверка:

```bash
make preflight
```

## Артефакты

- `build/integration-data-visibility-report.md`
- `build/integration-data-visibility-report.csv`

## Основное правило

Публичный репозиторий может содержать структуру, обезличенные сценарии, правила и документацию.

Реальные контакты сотрудников, данные клиентов, комментарии по сделкам и закрытые коммерческие условия должны храниться только в закрытом контуре.
