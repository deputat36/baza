# С чего начать просмотр проекта

## Коротко

Проект уже можно смотреть в GitHub как структуру будущей базы знаний.

Готовый рабочий инструмент для сотрудников появится после сборки закрытой Google Таблицы и переноса туда рабочих данных.

## Что открыть первым

1. `README.md` — общее описание.
2. `docs/intermediate-preview.md` — быстрый обзор того, что уже собрано.
3. `docs/project-index.md` — карта всех файлов проекта.
4. `docs/google-sheets-mvp.md` — как будет устроена первая Google Таблица.
5. `docs/google-sheet-launch-checklist.md` — чек-лист запуска закрытой таблицы.
6. `docs/google-apps-script-setup.md` — скрипты настройки таблицы.

## Что уже можно оценить без сборки

- Общую идею базы знаний.
- Структуру разделов.
- Логику карточек контактов, ситуаций и документов.
- Черновики городских и рабочих разделов.
- Правила безопасности и заполнения.
- План импорта в Google Таблицу.
- Apps Script для настройки закрытой таблицы.

## Где лежат черновики данных

```text
data/drafts/
data/dictionaries/
templates/
```

В этих файлах есть структура и безопасные черновики.

Реальные внутренние телефоны, ФИО, закрытые условия и клиентские данные в публичный репозиторий не добавляются.

## Как посмотреть промежуточный XLSX

Если работа идёт локально:

```bash
make install
make preflight
```

Ожидаемый файл:

```text
build/baza-knowledge-mvp.xlsx
```

Если сборка идёт через GitHub Actions, файл нужно искать в артефактах workflow:

```text
Actions -> Validate data files -> Artifacts -> baza-build-artifacts
```

## Как собрать закрытую Google Таблицу

1. Создать закрытую Google Таблицу.
2. Подключить Apps Script из папки `scripts/google-apps-script/`.
3. Обновить страницу таблицы.
4. Открыть меню `База знаний`.
5. Запустить `Полная подготовка таблицы`.
6. Импортировать CSV по `build/import-plan.md` или по `docs/import-manifest.md`.
7. Запустить `База знаний -> 3. Проверить данные`.
8. Исправить замечания на листе `Проверка данных`.

## Apps Script-файлы

```text
scripts/google-apps-script/setup-baza.gs
scripts/google-apps-script/setup_baza_sheet.gs
scripts/google-apps-script/validate-baza-sheet.gs
scripts/google-apps-script/menu.gs
```

## Что пока не готово

- Закрытая рабочая Google Таблица с реальными контактами.
- Проверенные реальные телефоны и ответственные лица.
- Полное наполнение городских организаций.
- Тестирование на сотрудниках.
- Финальный запуск на офис.

## Ближайший практический шаг

Собрать XLSX или закрытую Google Таблицу, открыть её глазами и проверить:

- понятны ли листы;
- удобно ли искать контакты;
- хватает ли городских разделов;
- какие контакты нужно дозаполнить вручную;
- какие поля лишние для сотрудников.
