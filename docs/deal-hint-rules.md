# Правила подсказок сделки

## Назначение

Правила подсказок сделки описывают, какие записи базы знаний нужно показывать сотруднику при определённых признаках сделки.

Это подготовка к будущей интеграции. Сейчас правила не подключены к Навигатору сделок и не меняют его поведение.

## Источник

```text
data/drafts/deal-hint-rules.csv
```

## Генератор

```text
scripts/tools/build_deal_hint_rules.py
```

## Артефакты

```text
build/deal-hint-rules.json
build/deal-hint-rules-report.md
build/deal-hint-rules-report.csv
```

## Запуск

```bash
make deal-hints
```

Или полный набор проверок:

```bash
make preflight
```

## Логика

Каждое правило связывает сигнал сделки с ID базы знаний.

Пример:

```text
has_children=true -> SIT-0001; DOC-0007
```

Это означает: если в будущей карточке сделки будет признак `has_children=true`, интерфейс сможет показать сотруднику ситуацию по детским долям и документ по разрешению органов опеки.

## Проверяемые словари

Генератор проверяет, что:

- сигнал из поля `Сигнал сделки` есть в `data/dictionaries/deal-signals.csv`;
- аудитории из поля `Кому показывать` есть в `data/dictionaries/deal-audiences.csv`;
- связанные ID существуют в `build/knowledge-index.json`.

## Важные ограничения

- Правила не должны содержать реальные данные сделок.
- Правила не должны содержать ФИО клиентов и телефоны участников.
- Все связанные ID должны существовать в `knowledge-index.json`.
- Все аудитории должны быть из контролируемого словаря.
- Правила являются read-only контрактом для будущего интерфейса.

## Практический смысл

Такой слой позволит в будущем показывать в карточке сделки не всю базу знаний, а только применимые подсказки:

- документы;
- типовые ситуации;
- контакты;
- частые ошибки;
- порядок действий.
