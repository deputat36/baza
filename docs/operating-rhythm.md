# Регламент эксплуатации базы знаний

## Назначение

Регламент эксплуатации показывает, что нужно регулярно проверять после запуска базы в офисе.

Он нужен, чтобы база знаний оставалась рабочей: контакты не устаревали, предложения сотрудников не терялись, а критичные разделы имели владельцев.

## Основной файл

`data/drafts/operating-rhythm.csv`

## Отчет

```bash
python scripts/tools/build_operating_rhythm_report.py
```

Результат:

- `build/operating-rhythm-report.md`
- `build/operating-rhythm-report.csv`

## Как использовать

1. Открыть отчет.
2. Смотреть задачи по периодам: ежедневно, еженедельно, ежемесячно, ежеквартально.
3. Назначить реальных ответственных в закрытой Google Таблице.
4. Фиксировать выполнение задач, которые блокируют стабильность базы.
5. После изменения структуры запускать полный `make preflight`.

## Важно

В публичном репозитории указываются только роли и безопасные шаблоны.

Реальные ФИО, телефоны сотрудников, клиентские данные и внутренние комментарии должны храниться только в закрытой рабочей таблице.
