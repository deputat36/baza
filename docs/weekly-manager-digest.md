# Еженедельный дайджест менеджера

## Назначение

Еженедельный дайджест нужен, чтобы менеджер видел в одном месте:

- есть ли блокеры запуска;
- какое решение по запуску зафиксировано;
- какие метрики использования требуют внимания;
- какие действия плана улучшений открыты;
- сколько предложений сотрудников ждут разбора;
- какие разделы требуют проверки актуальности.

## Генератор

`scripts/tools/build_weekly_manager_digest.py`

## Результаты

После запуска появляются файлы:

- `build/weekly-manager-digest.md` — короткая управленческая сводка;
- `build/weekly-manager-digest.csv` — табличная версия сводки.

## Запуск

```bash
make weekly-digest
```

Или в составе общей проверки:

```bash
make preflight
```

## Источники

Дайджест использует уже собранные отчеты из `build/`:

- `go-no-go-report.csv`;
- `launch-decision-log.csv`;
- `usage-metrics-report.csv`;
- `adoption-improvement-plan.csv`;
- `change-request-report.csv`;
- `data-freshness-report.csv`.

Если источники не найдены, дайджест не падает, а показывает замечание и предлагает запустить `make preflight`.

## Как использовать

Раз в неделю менеджер открывает `weekly-manager-digest.md`, смотрит строки со статусами `WARN`, `CHECK` или `PENDING`, назначает ответственных и фиксирует действия в закрытой рабочей таблице.

## Ограничение

Дайджест не подтверждает актуальность реальных контактов и юридическую точность данных. Он нужен для регулярного управленческого контроля.
