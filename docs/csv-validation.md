# Проверка CSV-файлов

## Назначение

Перед импортом в Excel или Google Таблицу нужно проверить CSV-файлы на базовые ошибки.

Проверка помогает найти:

- пустые файлы;
- строки без ID;
- дубли ID;
- строки с малым количеством столбцов;
- проблемы с заголовком.

## Скрипт

`scripts/tools/validate_csv_structure.py`

## Запуск

```bash
pip install -r requirements.txt
python scripts/tools/validate_csv_structure.py
```

Если нужно получить только отчёт без ошибки завершения:

```bash
python scripts/tools/validate_csv_structure.py --warn-only
```

## Поведение в CI

Без флага `--warn-only` скрипт завершится с кодом `1`, если найдёт структурные проблемы.

Это нужно, чтобы GitHub Actions не пропускал повреждённые CSV-файлы.

## Что делать с замечаниями

Если скрипт нашёл проблему:

1. Открыть указанный CSV.
2. Проверить строку.
3. Исправить ID или структуру.
4. Повторить запуск проверки.
5. После исправления импортировать данные в рабочую таблицу.

## Важно

Скрипт не проверяет юридическую точность данных и актуальность контактов.

Он проверяет только техническую структуру CSV.
