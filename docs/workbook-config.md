# Конфигурация XLSX-сборки

## Назначение

Файл `scripts/tools/workbook_config.py` хранит общую карту XLSX-сборки:

- название выходного файла;
- каталог сборки;
- список листов;
- CSV-источники для каждого листа.

## Почему это отдельный файл

Раньше карта листов находилась внутри `build_workbook_from_csv.py`.

Из-за этого проверка источников импортировала XLSX-генератор и зависела от `openpyxl`, хотя для проверки наличия CSV эта библиотека не нужна.

Теперь:

- `build_workbook_from_csv.py` отвечает за создание XLSX;
- `validate_workbook_sources.py` отвечает за проверку источников;
- `workbook_config.py` хранит общие настройки без внешних зависимостей.

## Учебные источники

После аудита обучения в конфигурацию добавлены:

### Лист `Справочники`

- `data/dictionaries/training-stages.csv`;
- `data/dictionaries/training-competencies.csv`.

### Лист `Обучение`

- `data/drafts/training-learning-paths.csv`;
- `data/drafts/training-modules.csv`;
- `data/drafts/training-assessments.csv`;
- `data/drafts/training-materials-registry.csv`;
- существующие чек-листы, сценарии, приемочные тесты и план обучения работе с базой.

Это позволяет видеть новую учебную архитектуру в XLSX, HTML-превью и плане импорта без отдельного генератора.

## Как добавить CSV в XLSX

1. Открыть `scripts/tools/workbook_config.py`.
2. Найти нужный лист в `SHEETS`.
3. Добавить путь к CSV.
4. Запустить `make sources`.
5. Запустить `make xlsx` или `make preflight`.
6. Открыть XLSX и проверить результат глазами пользователя.

## Важно

Не добавляйте в конфигурацию приватные CSV с реальными контактами, данными клиентов, индивидуальными результатами сотрудников, внутренними условиями или ссылками на закрытые доказательства.

Исходные презентации, видео, PDF и рабочие тетради не включаются в публичную XLSX-сборку. Для них используется безопасный реестр `training-materials-registry.csv`.
