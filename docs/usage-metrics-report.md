# Отчет по использованию базы

## Назначение

Отчет по использованию показывает, приносит ли база знаний практическую пользу после запуска.

Он помогает увидеть:

- сколько сотрудников пользуется базой;
- сколько поисков выполняется;
- сколько запросов не дают результата;
- сколько предложений и замечаний появляется;
- закрываются ли замечания;
- какие роли требуют дополнительного обучения.

## Команды

```bash
make usage-metrics
```

Или напрямую:

```bash
python scripts/tools/build_usage_metrics_report.py
```

## Результат

- `build/usage-metrics-report.md`
- `build/usage-metrics-report.csv`

## Источник

- `data/drafts/usage-metrics-log.csv`

## Правило чтения

- `OK` — показатель достиг цели.
- `CHECK` — показатель есть, но цель не достигнута.
- `WARN` — показатель требует внимания менеджера.

## Важно

Реальные значения должны храниться в закрытой Google Таблице или внутреннем инструменте.

В публичный репозиторий нельзя переносить клиентские данные, ФИО клиентов, телефоны и детали сделок.
