# База знаний специалиста по недвижимости | Этажи Борисоглебск

Проект для создания внутренней базы знаний офиса недвижимости в Борисоглебске.

Цель проекта — собрать в одном месте рабочие контакты, инструкции, документы, алгоритмы сделок, локальную практику и полезные материалы, которые помогают специалистам быстрее и качественнее проводить сделки.

## Проверка и сборка

Перед переносом данных в закрытую Google Таблицу можно запустить технические проверки и собрать промежуточный XLSX:

```bash
make install
make preflight
```

Для отдельной проверки обязательных CSV-источников XLSX:

```bash
make sources
```

## Будущая совместимость

Проект пока не совмещается с Навигатором сделок.

Но структура развивается так, чтобы в будущем база знаний могла подключаться к карточке сделки как read-only источник подсказок: через стабильные ID, связанные записи, управленческую сводку, GO/NO-GO решение запуска, журнал решения запуска, метрики использования, план улучшений после запуска, еженедельный дайджест менеджера, реестр управленческих действий, маршрутизацию к департаментам центрального офиса, схему закрытого реестра контактов, регламент проверки актуальности контактов, навигацию по ролям, ответственность за разделы, приемочные сценарии офиса, обучение сотрудников, пакет запуска, чек-лист запуска в офисе, регламент эксплуатации после запуска, JSON-индекс, отчёт связей, контроль актуальности данных, очередь предложений и исправлений, словарь сигналов, правила подсказок, аудитории подсказок, карту UI-зон, примеры payload, проверку видимости данных, матрицу доступа, проверку JSON-полей, контракты интеграции и безопасные тестовые сценарии.

Рабочие данные сделок, клиентские данные и внутренние комментарии не должны переноситься в публичный репозиторий.

## Важное правило безопасности

Репозиторий не должен содержать персональные данные клиентов, приватные телефоны сотрудников, внутренние коммерческие условия, доступы, токены, пароли и конфиденциальные документы.

Если в проект будут добавляться рабочие контакты сотрудников, внутренние материалы компании или приватные комментарии, репозиторий необходимо сделать приватным.

## Что хранится в репозитории

- архитектура базы знаний;
- правила заполнения;
- шаблоны таблиц;
- структура будущего CRM-модуля;
- дорожная карта;
- история изменений.

## Где ведутся рабочие данные

На первом этапе рабочие данные удобнее вести в закрытой Google Таблице. Репозиторий используется как место хранения структуры, правил и версий проекта.
