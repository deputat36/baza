# База знаний специалиста по недвижимости | Этажи Борисоглебск

Проект для создания внутренней базы знаний и единой системы обучения офиса недвижимости в Борисоглебске.

Цель проекта — собрать в одном месте инструкции, ситуации, документы, локальную практику, ролевые маршруты и безопасные учебные структуры, которые помогают специалистам быстрее осваивать профессию и качественнее проводить сделки.

Операционные действия и факты работы ведутся в `Космосе`. Публичный репозиторий хранит только безопасную структуру, правила, шаблоны и обезличенные примеры.

## Единая система обучения

Проект поддерживает маршруты:

1. Кандидат и preboarding.
2. Первый день.
3. Быстрый старт 30 дней.
4. СПН 2–3 месяца.
5. СПН 4–6 месяцев.
6. СПН 6+ месяцев.
7. Кандидат в наставники.
8. МОП и руководитель.
9. EVP и удержание.

Основные файлы:

- `data/dictionaries/training-stages.csv` — этапы обучения;
- `data/dictionaries/training-competencies.csv` — единый словарь компетенций;
- `data/drafts/training-learning-paths.csv` — маршруты;
- `data/drafts/training-modules.csv` — практические модули;
- `data/drafts/training-assessments.csv` — контрольные точки;
- `data/drafts/training-materials-registry.csv` — безопасный реестр мастер-материалов.

Главный принцип:

```text
роль -> этап -> компетенция -> материал -> практика -> действие в Космосе -> подтверждение -> следующий этап
```

Фактические действия не переписываются вручную в учебные таблицы, если они уже отражены в `Космосе`.

## Проверка и сборка

Перед переносом данных в закрытую Google Таблицу можно запустить технические проверки и собрать промежуточный XLSX:

```bash
make install
make preflight
```

Для отдельной проверки обязательных CSV-источников XLSX:

```bash
make sources
```

Для отдельной проверки согласованности Makefile, workflow и генераторов:

```bash
make pipeline-health
```

## Будущая совместимость

Проект пока не совмещается с Навигатором сделок.

Структура развивается как read-only источник знаний и подсказок. Операционные данные клиентов, объектов, действий и сделок остаются в соответствующей рабочей системе и не переносятся в публичную базу.

## Важное правило безопасности

Репозиторий не должен содержать персональные данные клиентов, приватные телефоны сотрудников, результаты конкретных сотрудников, внутренние коммерческие условия, доступы, токены, пароли и конфиденциальные документы.

Полные тексты внутренних презентаций, рабочих тетрадей и материалов с ограничением копирования должны храниться только в закрытом Drive. В GitHub сохраняются безопасные метаданные и собственные структурированные выводы.

## Что хранится в репозитории

- архитектура базы знаний и обучения;
- правила заполнения;
- шаблоны таблиц;
- безопасные ролевые маршруты;
- словари этапов и компетенций;
- дорожная карта;
- история изменений.

## Где ведутся рабочие данные

Операционные факты ведутся в `Космосе`. Реальные контакты, индивидуальные результаты обучения и внутренние материалы хранятся в закрытых Google-ресурсах. Репозиторий используется для безопасной структуры, правил и версий проекта.

## Принцип развития

Сначала проверяем базу знаний и учебные маршруты на небольшой группе. Затем упрощаем их по фактическому использованию. Только после подтверждения пользы принимается решение о закрытом портале или дополнительной интеграции.
