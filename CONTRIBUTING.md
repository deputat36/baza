# Как предлагать изменения

## Главное правило

Репозиторий публичный. Не добавляйте сюда реальные внутренние телефоны, данные клиентов, закрытые условия, пароли, токены и документы по сделкам.

## Что можно добавлять

- структуру разделов;
- обезличенные шаблоны;
- безопасные черновики;
- инструкции;
- правила заполнения;
- технические скрипты;
- предложения по улучшению базы.

## Что нельзя добавлять

- персональные данные клиентов;
- паспортные данные;
- сканы документов;
- внутренние контакты сотрудников;
- закрытые коммерческие условия;
- приватные комментарии по людям;
- доступы и токены.

## Как оформить изменение

1. Понять, к какому разделу относится правка.
2. Проверить, нет ли дубля.
3. Использовать существующий формат ID.
4. Поставить статус `DRAFT` или `CHECK`, если данные не проверены.
5. Добавить источник или комментарий.
6. Проверить CSV перед отправкой.

## Проверка перед публикацией

Перед отправкой изменений запустить:

```bash
pip install -r requirements.txt
python scripts/tools/validate_csv_structure.py
```

## Реальные рабочие данные

Реальные рабочие контакты и проверенные комментарии вносятся только в закрытую Google Таблицу.
